THIS ROUTINE CAN BE CALLED AT ANY TIME AFTER A BUFR MESSAGE HAS BEEN READ INTO MEMORY VIA SUBROUTINE READMG, READERME OR EQUIVALENT, AND IT CAN BE CALLED FOR ANY CODE OR FLAG TABLE MNEMONIC DEFINED WITHIN THAT MESSAGE.

IT SEARCHES FOR THE SPECIFIED MNEMONIC AND ASSOCIATED VALUE (CODE FIGURE OR BIT NUMBER) WITHIN THE INTERNAL MEMORY STRUCTURE FOR STORING CODE/FLAG TABLE INFORMATION, AND IF FOUND RETURNS THE ASSOCIATED MEANING AS A CHARACTER STRING. THE SEARCH MAY ALSO OPTIONALLY INCLUDE A SPECIFIED SECOND MNEMONIC AND ASSOCIATED VALUE UPON WHICH THE FIRST MNEMONIC AND ITS ASSOCIATED VALUE DEPEND, FOR CASES SUCH AS, E.G. WHEN THE MEANING OF AN ORIGINATING SUBCENTER VALUE DEPENDS ON THE IDENTITY OF THE ORIGINATING CENTER.

NOTE THAT THIS ROUTINE CAN ONLY BE CALLED FOR MNEMONICS WHICH ARE DEFINED WITHIN THE MESSAGE THAT WAS MOST RECENTLY READ INTO MEMORY. IN MOST CASES THIS MEANS THAT THE MNEMONIC MUST BE CONTAINED WITHIN THE SUBSET DEFINITION (SECTION 3) OF THAT MESSAGE. THE ONLY EXCEPTIONS TO THIS RULE ARE FOR ORIGINATING CENTERS, ORIGINATING SUBCENTERS, DATA TYPES AND DATA SUBTYPES, SINCE THOSE CAN ALSO BE CONTAINED WITHIN THE IDENTIFICATION SECTION (SECTION 1) OF A BUFR MESSAGE. IN ANY CASE, IF THE SEARCH IS UNSUCCESSFUL, AND IF THERE WAS NO OPTIONAL SECOND MNEMONIC AND ASSOCIATED VALUE SPECIFIED ON INPUT, THE ROUTINE WILL RE-SEARCH THE TABLE TO CHECK WHETHER THE MEANING OF THE FIRST MNEMONIC AND ASSOCIATED VALUE MAY INDEED DEPEND ON THE VALUE OF ONE OR MORE OTHER POSSIBLE SECOND MNEMONICS. IF SO, THOSE POSSIBLE MNEMONICS ARE RETURNED ALONG WITH A SPECIAL RETURN CODE SO THAT THE CALLING ROUTINE MAY EXAMINE THEM AND POSSIBLY ISSUE ANOTHER SUBSEQUENT CALL TO THIS SAME ROUTINE WITH SPECIFIED VALUES FOR THE SECOND MNEMONIC AND ASSOCIATED VALUE.

PROGRAM HISTORY LOG: 2018-01-11 J. ATOR – ORIGINAL AUTHOR 2018-01-11 J. ATOR – ADD SPECIAL HANDLING FOR DATA TYPES AND SUBTYPES IN SECTION 1

USAGE: CALL GETCFMNG ( LUNIT, NEMOI, IVALI, NEMOD, IVALD, CMEANG, LNMNG, IRET )

INPUT ARGUMENT LIST: LUNIT - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR BUFR FILE NEMOI - CHARACTER*(*): MNEMONIC TO SEARCH FOR IVALI - INTEGER: VALUE (CODE FIGURE OR BIT NUMBER) ASSOCIATED WITH NEMOI NEMOD - CHARACTER*(*): OPTIONAL SECOND MNEMONIC UPON WHICH THE VALUES NEMOI AND IVALI DEPEND; SET TO ALL BLANK CHARACTERS IF THE MEANINGS OF NEMOI AND IVALI DO NOT DEPEND ON THE VALUE OF ANY OTHER MNEMONIC IVALD - INTEGER: VALUE (CODE FIGURE OR BIT NUMBER) ASSOCIATED WITH NEMOD; SET TO (-1) WHENEVER NEMOD IS SET TO ALL BLANK CHARACTERS

OUTPUT ARGUMENT LIST: CMEANG - CHARACTER*(*): IF THE INITIAL SEARCH OF THE TABLE WAS SUCCESSFUL, THEN THIS STRING CONTAINS THE MEANING CORRESPONDING TO NEMOI AND IVALI (AND TO NEMOD AND IVALD, IF SPECIFIED). HOWEVER, IF THE INITIAL SEARCH WAS UNSUCCESSFUL, AND IF NO OPTIONAL SECOND MNEMONIC AND ASSOCIATED VALUE WERE SPECIFIED ON INPUT, AND IF THE SECOND SEARCH OF THE TABLE DETERMINED THAT THE MEANING OF THE FIRST MNEMONIC AND ASSOCIATED VALUE INDEED DEPENDS ON ONE OR MORE OTHER POSSIBLE SECOND MNEMONICS, THEN THOSE POSSIBLE SECOND MNEMONICS ARE RETURNED WITHIN THIS STRING, AS A SERIES OF IRET SUCCESSIVE 8-BYTE SUBSTRINGS LNMNG - INTEGER: LENGTH OF STRING RETURNED IN MEANING IRET - RETURN CODE: 0 = MEANING FOUND AND STORED IN CMEANG STRING -1 = MEANING NOT FOUND >0 = MEANING NOT FOUND, AND NEMOD AND IVALD WERE NOT SPECIFIED ON INPUT, AND THE MEANING OF NEMOI AND IVALI DEPENDS ON THE VALUE OF ONE OF THE MNEMONICS STORED IN THE FIRST IRET 8-BYTE SUBSTRINGS OF THE CMEANG STRING

REMARKS: THIS ROUTINE CALLS: BORT IFXY IREADMT NEMTAB NUMTBD PARSTR SRCHTBF STATUS THIS ROUTINE IS CALLED BY: None Normally called only by application programs.
