4 C> THIS SUBROUTINE READS INFORMATION FROM A BUFR DATA MESSAGE
5 C> ALREADY IN MEMORY, PASSED IN AS AN INPUT ARGUMENT. IT IS SIMILAR
6 C> TO BUFR ARCHIVE LIBRARY SUBROUTINE READMG EXCEPT, INSTEAD OF
7 C> READING BUFR MESSAGES DIRECTLY FROM A BUFR FILE THAT IS PHYSICALLY
8 C> STORED ON THE LOCAL SYSTEM AND INTERFACED TO THE SOFTWARE VIA A
9 C> LOGICAL UNIT NUMBER, IT READS BUFR MESSAGES DIRECTLY FROM A MEMORY
10 C> ARRAY WITHIN THE APPLICATION PROGRAM ITSELF. THIS PROVIDES USERS
11 C> WITH GREATER FLEXIBILITY FROM AN INPUT/OUTPUT PERSPECTIVE.
12 C> READERME CAN BE USED IN ANY CONTEXT IN WHICH READMG MIGHT OTHERWISE
13 C> BE USED. IF THIS MESSAGE IS NOT A BUFR MESSAGE, THEN AN
14 C> APPROPRIATE CALL IS MADE TO BUFR ARCHIVE LIBRARY SUBROUTINE BORT.
