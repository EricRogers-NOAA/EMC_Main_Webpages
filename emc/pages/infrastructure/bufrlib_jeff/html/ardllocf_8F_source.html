4 C> THIS SUBROUTINE FREES ANY MEMORY THAT WAS DYNAMICALLY
5 C> ALLOCATED BY PREVIOUS CALLS TO BUFR ARCHIVE LIBRARY ROUTINES
6 C> ARALLOCF OR ARALLOCC.
7 C>
8 C> NOTE THAT THIS SUBROUTINE IS CALLED WITHIN BUFR ARCHIVE LIBRARY
9 C> SUBROUTINE EXITBUFR AS PART OF THE PROCESS TO RESET THE LIBRARY AND
10 C> PREPARE IT FOR POTENTIAL RE-ALLOCATION OF NEW ARRAY SPACE VIA ONE OR
11 C> MORE SUBSEQUENT CALLS TO SUBROUTINES ISETPRM AND OPENBF. THIS
12 C> SUBROUTINE SHOULD ONLY BE CALLED DIRECTLY BY AN APPLICATION PROGRAM
13 C> IF THE PROGRAM IS COMPLETELY FINISHED WITH ALL CALLS TO ALL OTHER
14 C> BUFR ARCHIVE LIBRARY ROUTINES, BECAUSE THE MEMORY FREED HEREIN WILL
15 C> RENDER THE LIBRARY AS EFFECTIVELY UNUSABLE FOR THE REMAINDER OF THE
16 C> LIFE OF THE APPLICATION PROGRAM. HOWEVER, THIS MAY BE A USEFUL
17 C> OPTION FOR APPLICATION PROGRAMS WHICH WANT TO MOVE ON TO OTHER
18 C> UNRELATED TASKS WITHOUT CONTINUING TO TIE UP A SIGNIFICANT AMOUNT
19 C> OF DYNAMICALLY-ALLOCATED HEAP MEMORY RELATED TO THIS LIBRARY.
20 C> OTHERWISE, ALL SUCH MEMORY WILL BE FREED AUTOMATICALLY ONCE THE
21 C> APPLICATION PROGRAM TERMINATES.
22 C>
23 C> PROGRAM HISTORY LOG:
24 C> 2014-12-04 J. ATOR -- ORIGINAL AUTHOR
25 C> 2019-05-09 J. ATOR -- MODIFIED DEALLOCATIONS FOR MODA_BUFRMG
26 C>
27 C> USAGE: CALL ARDLLOCF
28 C>
29 C> REMARKS:
30 C> THIS ROUTINE CALLS: ARDLLOCC
31 C> THIS ROUTINE IS CALLED BY: EXITBUFR
32 C> Also called by application programs.
