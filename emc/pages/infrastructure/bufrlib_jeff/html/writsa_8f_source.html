4 C> THIS SUBROUTINE SHOULD ONLY BE CALLED WHEN LOGICAL UNIT
5 C> ABS(LUNXX) HAS BEEN OPENED FOR OUTPUT OPERATIONS.
6 C>
7 C> WHEN LUNXX IS GREATER THAN ZERO, IT PACKS UP THE CURRENT SUBSET
8 C> WITHIN MEMORY AND THEN TRIES TO ADD IT TO THE BUFR MESSAGE THAT IS
9 C> CURRENTLY OPEN WITHIN MEMORY FOR ABS(LUNXX). THE DETERMINATION AS
10 C> TO WHETHER OR NOT THE SUBSET CAN BE ADDED TO THE MESSAGE IS MADE
11 C> VIA AN INTERNAL CALL TO ONE OF THE BUFR ARCHIVE LIBRARY SUBROUTINES
12 C> WRCMPS OR MSGUPD, DEPENDING UPON WHETHER OR NOT THE MESSAGE IS
13 C> COMPRESSED. IF IT TURNS OUT THAT THE SUBSET CANNOT BE ADDED TO THE
14 C> CURRENTLY OPEN MESSAGE, THEN THAT MESSAGE IS FLUSHED TO ABS(LUNXX)
15 C> AND A NEW ONE IS CREATED IN ORDER TO HOLD THE SUBSET. AS LONG AS
16 C> LUNXX IS GREATER THAN ZERO, WRITSA FUNCTIONS EXACTLY LIKE BUFR
17 C> ARCHIVE LIBRARY SUBROUTINE WRITSB, EXCEPT THAT WRITSA ALSO RETURNS
18 C> A COPY OF EACH COMPLETED BUFR MESSAGE TO THE APPLICATION PROGRAM
19 C> IN THE FIRST MSGL WORDS OF ARRAY MSGT.
20 C>
21 C> ALTERNATIVELY, WHEN LUNXX IS LESS THAN ZERO, THIS IS A SIGNAL TO
22 C> FORCE ANY CURRENT MESSAGE IN MEMORY TO BE FLUSHED TO ABS(LUNXX) AND
23 C> RETURNED IN ARRAY MSGT. IN SUCH CASES, ANY CURRENT SUBSET IN MEMORY
24 C> IS IGNORED. THIS OPTION IS NECESSARY BECAUSE, IN MOST CASES, ANY
25 C> MESSAGE RETURNED IN MSGT FROM A CALL TO THIS ROUTINE DOESN'T CONTAIN
26 C> THE ACTUAL SUBSET THAT WAS PACKED UP AND STORED DURING THE SAME CALL
27 C> TO THIS ROUTINE. THEREFORE, THE ONLY WAY TO ENSURE THAT EVERY LAST
28 C> BUFR SUBSET IS RETURNED WITHIN A BUFR MESSAGE IN MSGT BEFORE, E.G.,
29 C> EXITING THE APPLICATION PROGRAM, IS TO DO ONE FINAL CALL TO THIS
30 C> ROUTINE WITH LUNXX LESS THAN ZERO IN ORDER TO FORCIBLY FLUSH OUT AND
31 C> RETURN ONE FINAL BUFR MESSAGE.
32 C>

96 
97 C----------------------------------------------------------------------
98 C----------------------------------------------------------------------
99 
100  lunit = abs(lunxx)
101 
102 C CHECK THE FILE STATUS
103 C ---------------------
104 
105  CALL status(lunit,lun,il,im)
106  IF(il.EQ.0) goto 900
107  IF(il.LT.0) goto 901
108  IF(im.EQ.0) goto 902
109 
110 C IF LUNXX < 0, FORCE MEMORY MSG TO BE WRITTEN (W/O ANY CURRENT SUBSET)
111 C ---------------------------------------------------------------------
112 
113  IF(lunxx.LT.0) CALL closmg(lunit)
114 
115 C IS THERE A COMPLETED BUFR MESSAGE TO BE RETURNED?
116 C -------------------------------------------------
117 
118  IF(msglen(lun).GT.0) THEN
119  IF(msglen(lun).GT.lmsgt) goto 904
120  msgl = msglen(lun)
121  DO n=1,msgl
122  msgt(n) = msgtxt(n,lun)
123  ENDDO
124  msglen(lun) = 0
125  ELSE
126  msgl = 0
127  ENDIF
128 
129  IF(lunxx.LT.0) goto 100
130 
131 C PACK UP THE SUBSET AND PUT IT INTO THE MESSAGE
132 C ----------------------------------------------
133 
134  CALL wrtree(lun)
135  IF( ccmf.EQ.'Y' ) THEN
136  CALL wrcmps(lunit)
137  ELSE
138  CALL msgupd(lunit,lun)
139  ENDIF
140 
141 C IF THE JUST-COMPLETED CALL TO WRCMPS OR MSGUPD FOR THIS SUBSET CAUSED
142 C A MESSAGE TO BE FLUSHED TO ABS(LUNXX), THEN ATTEMPT TO RETRIEVE AND
143 C RETURN THAT MESSAGE NOW. OTHERWISE, WE RUN THE RISK THAT THE NEXT
144 C CALL TO OPENMB OR OPENMG MIGHT CAUSE ANOTHER MESSAGE TO BE FLUSHED,
145 C AND THUS OVERWRITE THE CURRENT MESSAGE WITHIN ARRAY MSGTXT BEFORE WE
146 C HAD THE CHANCE TO RETRIEVE IT DURING THE NEXT CALL TO WRITSA.
147 
148 C ALSO NOTE THAT, IN RARE INSTANCES (E.G. IF THE BYTE COUNT OF THE MOST
149 C RECENT SUBSET IS > 65530), WE COULD END UP WITH TWO BUFR MESSAGES
150 C AVAILABLE TO BE RETURNED FROM THIS ONE CALL TO WRITSA. IF SUFFICIENT
151 C SPACE IS AVAILABLE IN THE MSGT ARRAY, THEN GO AHEAD AND RETURN BOTH
152 C MESSAGES NOW.
153 
154  IF( (msglen(lun).GT.0) .AND. (msgl+msglen(lun).LE.lmsgt) ) THEN
155  DO n = 1,msglen(lun)
156  msgt(msgl+n) = msgtxt(n,lun)
157  ENDDO
158  msgl = msgl+msglen(lun)
159  msglen(lun) = 0
160  ENDIF
161 
162 C EXITS
163 C -----
164 
165 100 RETURN

167  . 'MUST BE OPEN FOR OUTPUT')
168 901 CALL bort('BUFRLIB: WRITSA - OUTPUT BUFR FILE IS OPEN FOR '//
169  . 'INPUT, IT MUST BE OPEN FOR OUTPUT')
170 902 CALL bort('BUFRLIB: WRITSA - A MESSAGE MUST BE OPEN IN OUTPUT '//

172 904 CALL bort('BUFRLIB: WRITSA - OVERFLOW OF OUTPUT BUFR MESSAGE '//
173  . 'ARRAY; TRY A LARGER DIMENSION FOR THIS ARRAY')
174  END
