4 C> THIS SUBROUTINE OPENS A BUFR FILE FOR INPUT, READS EACH
5 C> MESSAGE AND TRANSFERS THEM ONE-BY-ONE TO INTERNAL MEMORY (ARRAY
6 C> MSGS IN MODULE MSGMEM). IF MESSAGES ARE APPENDED TO EXISTING
7 C> MESSAGES IN INTERNAL MEMORY, THE BUFR FILE READ HERE IS
8 C> CLOSED PRIOR TO RETURNING TO THE CALLING PROGRAM. AN ARRAY IS
9 C> ALSO RETURNED CONTAINING A LIST OF MESSAGE TYPES READ IN.
10 C>
11 C> THIS IS A VARIATION OF UFBMEM WHICH ENABLES MESSAGE SORTING BEFORE
12 C> READING. BECAUSE OF THIS RE-ORDERING, EMBEDDED TABLE MESSAGES ARE
13 C> NOT STORED IN MODULE MSGMEM, SINCE THEY ARE NO LONGER RELEVANT
14 C> ONCE THE RE-ORDERING (I.E. SORTING) HAS TAKEN PLACE. INSTEAD, A
15 C> SEPARATE UNIT NUMBER IS ADDED TO THE INPUT ARGUMENTS TO SPECIFY
16 C> WHERE THE NECESSARY BUFR TABLE INFORMATION CAN BE FOUND.
17 C>
18 C> PROGRAM HISTORY LOG:
19 C> 2012-01-26 J. WOOLLEN -- MODIFIED UFBMEM TO READ AND SORT MEMORY
20 C> MESSAGES FOR TRANJB INGEST ROUTINES AND
21 C> RETURN A LIST OF MESSAGE TYPES READ IN.
22 C> ALSO, A SEPARATE INPUT ARGUMENT IS ADDED
23 C> TO SPECIFY WHERE TO FIND THE BUFR TABLE,
24 C> INSTEAD OF SAVING EMBEDDED DICTIONARY
25 C> MESSAGES IN MODULE MSGMEM
26 C> 2014-12-10 J. ATOR -- USE MODULES INSTEAD OF COMMON BLOCKS
27 C> 2015-09-24 D. STOKES -- FIX MISSING DECLARATION OF COMMON /QUIET/
28 C>
29 C> USAGE: CALL UFBMEX (LUNIT, LUNDX, INEW, IRET, MESG)
30 C> INPUT ARGUMENT LIST:
31 C> LUNIT - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR BUFR FILE
32 C> LUNDX - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR USER-
33 C> SUPPLIED BUFR DICTIONARY TABLE IN CHARACTER FORMAT
34 C> INEW - INTEGER: SWITCH:
35 C> 0 = initialize internal arrays prior to

38 C> internal memory arrays
39 C>
40 C> OUTPUT ARGUMENT LIST:

44 C> INPUT FILES:
