4 C> THIS SUBROUTINE PARSES A USER-SPECIFIED TAG (MNEMONIC)
5 C> (UTG) THAT REPRESENTS A VALUE EITHER BEING DECODED FROM A BUFR FILE
6 C> (IF IT IS BEING READ) OR ENCODED INTO A BUFR FILE (IF IT IS BEING
7 C> WRITTEN). THIS SUBROUTINE FIRST CHECKS TO SEE IF THE TAG CONTAINS
8 C> A CONDITION CHARACTER ('=', '!', '<', '>', '^' OR '#'). IF IT DOES
9 C> NOT, NOTHING HAPPENS AT THIS POINT. IF IT DOES, THEN THE TYPE OF
10 C> CONDITION CHARACTER IS NOTED AND THE TAG IS STRIPPED OF ALL
11 C> CHARACTERS AT AND BEYOND THE CONDITION CHARACTER. IN EITHER EVENT,
12 C> THE RESULTANT TAG IS CHECKED AGAINST THOSE IN THE INTERNAL JUMP/
13 C> LINK SUBSET TABLE (IN MODULE TABLES). IF FOUND, THE NODE
14 C> ASSOCIATED WITH THE TAG IS RETURNED (AND IT IS EITHER A "CONDITION"
15 C> NODE OR A "STORE" NODE DEPENDING OF THE PRESENCE OR ABSENCE OF A
16 C> CONDITION CHARACTER IN UTG). OTHERWISE THE NODE IS RETURNED AS
17 C> ZERO. IF THE TAG REPRESENTS A CONDITION NODE, THEN THE CONDITION
18 C> VALUE (NUMERIC CHARACTERS BEYOND THE CONDITION CHARACTER IN THE
19 C> USER-SPECIFIED TAG INPUT HERE) IS RETURNED.
20 C>
21 C> AS AN EXAMPLE OF CONDITION CHARACTER USAGE, CONSIDER THE FOLLOWING
22 C> EXAMPLE OF A CALL TO UFBINT:
23 C>
24 C> REAL*8 USR(4,50)
25 C> ....
26 C> ....
27 C> CALL UFBINT(LUNIN,USR,4,50,IRET,'PRLC<50000 TMDB WDIR WSPD')
28 C>
29 C> ASSUMING THAT LUNIN POINTS TO A BUFR FILE OPEN FOR INPUT (READING),
30 C> THEN THE USR ARRAY NOW CONTAINS IRET LEVELS OF DATA (UP TO A MAXIMUM
31 C> OF 50!) WHERE THE VALUE OF PRLC IS/WAS LESS THAN 50000, ALONG WITH
32 C> THE CORRESPONDING VALUES FOR TMDB, WDIR AND WSPD AT THOSE LEVELS.
33 C>
34 C> AS ANOTHER EXAMPLE, CONSIDER THE FOLLOWING EXAMPLE OF A CALL TO
35 C> READLC FOR A LONG CHARACTER STRING:
36 C>
37 C> CHARACTER*200 LCHR
38 C> ....
39 C> ....
40 C> CALL READLC(LUNIN,LCHR,'NUMID#3')
41 C>
42 C> ASSUMING THAT LUNIN POINTS TO A BUFR FILE OPEN FOR INPUT (READING),
43 C> THEN THE LCHR STRING NOW CONTAINS THE VALUE CORRESPONDING TO THE
44 C> THIRD OCCURRENCE OF NUMID WITHIN THE CURRENT SUBSET.
45 C>
46 C> VALID CONDITION CODES INCLUDE:
47 C> '<' - LESS THAN
48 C> '>' - GREATER THAN
49 C> '=' - EQUAL TO
50 C> '!' - NOT EQUAL TO
51 C> '#' - ORDINAL IDENTIFIER FOR A PARTICULAR OCCURRENCE OF A LONG
52 C> CHARACTER STRING
53 C>

162 C PARSE UTG, SAVING INTO ATAG ONLY CHARACTERS PRIOR TO CONDITION CHAR.
163 C --------------------------------------------------------------------
164 
165 C But first, take care of the special case where UTG denotes the
166 C short (i.e. 1-bit) delayed replication of a Table D mnemonic.
167 C This will prevent confusion later on since '<' and '>' are each
168 C also valid as condition characters.
169 
170  IF((utg(1:1).EQ.'<').AND.(index(utg(3:),'>').NE.0)) THEN

173  ENDIF
174

182  ENDIF
183  ENDDO

185  ENDDO

189 
190 1 inod = inode(lun)

192  IF(atag.EQ.tag(nod)) goto 2
193  ENDDO
194 
195 C ATAG NOT FOUND IN SUBSET TABLE
196 C ------------------------------
197 
198 C So what do we want to do? We could be "picky" and abort right
199 C here, or we could allow for the possibility that, e.g. a user
200 C application has been streamlined to always call UFBINT with the
201 C same STR, even though some of the mnemonics contained within that
202 C STR may not exist within the sequence definition of every
203 C possible type/subtype that is being written by the application.
204 C In such cases, by not being "picky", we could just allow BUFRLIB
205 C to subsequently (and quietly, if IPRT happened to be set to -1
206 C in COMMON /QUIET/!) not actually store the value corresponding
207 C to such mnemonics, rather than loudly complaining and aborting.
208 
209  IF(kon.EQ.0 .AND. (io.EQ.0.OR.atag.EQ.'NUL'.OR..NOT.picky)) THEN
210 C i.e. (if this tag does not contain any condition characters)
211 C .AND.
212 C ((either the file is open for input) .OR.
213 C (the tag consists of 'NUL') .OR.
214 C (we aren't being "picky"))
215  nod = 0
216  goto 100
217  ELSE
218 C abort...
219  goto 900
220  ENDIF
221 
222 C ATAG IS FOUND IN SUBSET TABLE, MAKE SURE IT HAS A VALID NODE TYPE
223 C -----------------------------------------------------------------
224 
225 2 IF(kon.EQ.5) THEN
226 c .... Cond. char "^" must be assoc. with a delayed replication
227 c sequence (this is a "bump" node) (Note: This is obsolete but
228 c remains for "old" programs using the BUFR ARCHIVE LIBRARY)
229  IF(typ(nod-1).NE.'DRP' .AND. typ(nod-1).NE.'DRS') goto 901
230  ELSEIF(kon.NE.6) THEN
231 C Allow reading (but not writing) of delayed replication factors.
232  atyp = typ(nod)
233  DO i=1,nchk
234  IF(atyp.EQ.btyp(i) .AND. io.GT.iok(i)) goto 902
235  ENDDO
236  ENDIF
237 
238 C IF CONDITION NODE, GET CONDITION VALUE WHICH IS A NUMBER FOLLOWING IT
239 C ---------------------------------------------------------------------
240 
241  IF(kon.NE.0) THEN

243  IF(num.LT.0) goto 903
244  val = num
245  ENDIF
246
