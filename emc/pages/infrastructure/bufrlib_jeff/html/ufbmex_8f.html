THIS SUBROUTINE OPENS A BUFR FILE FOR INPUT, READS EACH MESSAGE AND TRANSFERS THEM ONE-BY-ONE TO INTERNAL MEMORY (ARRAY MSGS IN MODULE MSGMEM).

IF MESSAGES ARE APPENDED TO EXISTING MESSAGES IN INTERNAL MEMORY, THE BUFR FILE READ HERE IS CLOSED PRIOR TO RETURNING TO THE CALLING PROGRAM. AN ARRAY IS ALSO RETURNED CONTAINING A LIST OF MESSAGE TYPES READ IN.

THIS IS A VARIATION OF UFBMEM WHICH ENABLES MESSAGE SORTING BEFORE READING. BECAUSE OF THIS RE-ORDERING, EMBEDDED TABLE MESSAGES ARE NOT STORED IN MODULE MSGMEM, SINCE THEY ARE NO LONGER RELEVANT ONCE THE RE-ORDERING (I.E. SORTING) HAS TAKEN PLACE. INSTEAD, A SEPARATE UNIT NUMBER IS ADDED TO THE INPUT ARGUMENTS TO SPECIFY WHERE THE NECESSARY BUFR TABLE INFORMATION CAN BE FOUND.

PROGRAM HISTORY LOG: 2012-01-26 J. WOOLLEN – MODIFIED UFBMEM TO READ AND SORT MEMORY MESSAGES FOR TRANJB INGEST ROUTINES AND RETURN A LIST OF MESSAGE TYPES READ IN. ALSO, A SEPARATE INPUT ARGUMENT IS ADDED TO SPECIFY WHERE TO FIND THE BUFR TABLE, INSTEAD OF SAVING EMBEDDED DICTIONARY MESSAGES IN MODULE MSGMEM 2014-12-10 J. ATOR – USE MODULES INSTEAD OF COMMON BLOCKS 2015-09-24 D. STOKES – FIX MISSING DECLARATION OF COMMON /QUIET/

USAGE: CALL UFBMEX (LUNIT, LUNDX, INEW, IRET, MESG) INPUT ARGUMENT LIST: LUNIT - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR BUFR FILE LUNDX - INTEGER: FORTRAN LOGICAL UNIT NUMBER FOR USER- SUPPLIED BUFR DICTIONARY TABLE IN CHARACTER FORMAT INEW - INTEGER: SWITCH: 0 = initialize internal arrays prior to transferring messages here else = append the messages transferred here to internal memory arrays

OUTPUT ARGUMENT LIST: IRET - INTEGER: NUMBER OF MESSAGES TRANSFERRED MESG - INTEGER: ARRAY OF MESSAGE TYPES READ INTO MEMORY

INPUT FILES: UNIT "LUNIT" - BUFR FILE UNIT "LUNDX" - BUFR DICTIONARY TABLE IN CHARACTER FORMAT

REMARKS: NOTE THAT IREADMM, RDMEMM, READMM, UFBMMS, UFBMNS, UFBRMS, UFBTAB OR UFBTAM CAN BE CALLED AFTER THIS TO READ SPECIFIC BUFR MESSAGES FROM INTERNAL MEMORY.

THIS ROUTINE CALLS: BORT CLOSBF ERRWRT IUPBS01 NMWRD OPENBF RDMSGW THIS ROUTINE IS CALLED BY: None Normally called only by application programs.
