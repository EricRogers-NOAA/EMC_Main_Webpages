C C SUBPROGRAM: SRCHTBF C PRGMMR: ATOR ORG: NCEP DATE: 2018-01-11 C C ABSTRACT: THIS ROUTINE SEARCHES FOR A SPECIFIED DESCRIPTOR AND C ASSOCIATED VALUE (CODE FIGURE OR BIT NUMBER) WITHIN THE INTERNAL C MEMORY STRUCTURE FOR STORING CODE/FLAG TABLE INFORMATION.

THE C SEARCH MAY ALSO OPTIONALLY INCLUDE A SPECIFIED SECOND DESCRIPTOR C AND ASSOCIATED VALUE UPON WHICH THE FIRST DESCRIPTOR AND ITS C ASSOCIATED VALUE DEPEND, FOR CASES SUCH AS, E.G. WHEN THE MEANING C OF AN ORIGINATING SUBCENTER VALUE DEPENDS ON THE IDENTITY OF THE C ORIGINATING CENTER. C C IF THE REQUESTED ENTRY IN THE TABLE IS FOUND, THE ROUTINE RETURNS C THE ASSOCIATED MEANING AS A CHARACTER STRING. OTHERWISE, AND IF C THERE WAS NO OPTIONAL SECOND DESCRIPTOR AND ASSOCIATED VALUE C SPECIFIED ON INPUT, THE ROUTINE WILL RE-SEARCH THE TABLE TO CHECK C WHETHER THE MEANING OF THE FIRST DESCRIPTOR AND ASSOCIATED VALUE C MAY INDEED DEPEND ON THE VALUE OF ONE OR MORE OTHER POSSIBLE C SECOND DESCRIPTORS. void wrdlen ( void  )

THIS SUBROUTINE FIGURES OUT SOME IMPORTANT INFORMATION ABOUT THE LOCAL MACHINE ON WHICH THE BUFR ARCHIVE LIBRARY SOFTWARE IS BEING RUN AND STORES THIS INTO COMMON BLOCK /HRDWRD/.

SUCH INFORMATION INCLUDES DETERMINING THE NUMBER OF BITS AND THE NUMBER OF BYTES IN A MACHINE WORD AS WELL AS DETERMINING WHETHER THE MACHINE USES THE ASCII OR EBCDIC CHARACTER SET.

NOTE: IT IS ONLY NECESSARY FOR THIS SUBROUTINE TO BE CALLED ONCE, AND THIS IS NORMALLY DONE DURING THE FIRST CALL TO BUFR ARCHIVE LIBRARY SUBROUTINE OPENBF. HOWEVER, THE SUBROUTINE DOES KEEP TRACK OF WHETHER IT HAS ALREADY BEEN CALLED; THUS, IF IT IS CALLED AGAIN LATER BY A DIFFERENT BUFR ARCHIVE LIBRARY SUBROUTINE, IT WILL JUST QUIETLY RETURN WITHOUT (RE)COMPUTING ALL OF THE INFORMATION WITHIN COMMON BLOCK /HRDWRD/.
